# 代理模式

## 代理模式
为一个对象提供一个替身，以控制对这个对象的访问，即通过代理对象访问目标对象，这样做的好处是可以在目标对象实现的基础上，增强额外的功能操作，即扩展目标对象的功能



## 不同形式
- 静态代理
- 动态代理：jdk代理   接口代理
- Cglib代理：可以在内存动态的创建对象，不需要实现接口，树与动态代理的范畴

## 动态代理
1. 代理对象不需要实现接口，但是目标对象需要实现接口，否则不能用动态代理
2. 代理对象的生成，是利用jdk的API，动态的在内存中构建代理对象
3. 动态代理也叫做：jdk代理，接口代理

## 步骤
1. 利用传入的对象 teacherDao，目标对象
2. 利用反射机制，返回一个代理对象
3. 然后通过代理对象，调用目标对象方法
